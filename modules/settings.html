Settings & Configuration

System setup and organizational configuration

Module Overview

The Settings Page is the administrative hub where system administrators configure the entire Manpower Management System. This is where you define organizational structure, roles, permissions, and system parameters that affect all other modules.

Define all foundational system settings

Key Features

Configure organizational hierarchies and classifications

  • • Ranks (RK): Define military rank structure
  • • Companies (Coy): Set up organizational units
  • • Skills: Define professional skills taxonomy

Control user access and authorization levels

  • • Manage user status and deactivation

Best Practices

Do's

  • ✓ Set up all mandatory configurations before adding users
  • ✓ Use clear, descriptive names for ranks and companies
  • ✓ Regularly review and update user permissions
  • ✓ Create roles based on job functions
  • ✓ Test configurations before going live

Don'ts

  • ✗ Don't delete configurations that are already in use
  • ✗ Don't grant excessive permissions to users
  • ✗ Don't skip the initial setup process
  • ✗ Don't share admin credentials
  • ✗ Don't modify system settings without documentation

Frequently Asked Questions

Can I modify configurations after creating user accounts?

Yes, most configurations can be modified. However, ensure that changes don't break existing data relationships or user workflows. Always backup before making major changes.

How many ranks and companies can I create?

There's no hard limit, but it's recommended to keep the structure manageable. Typically, 5-15 ranks and 2-10 companies work well for most organizations.

What happens if I delete a rank that soldiers have?

The system will prevent deletion if the rank is in use. You must first reassign all soldiers with that rank to a different rank before deletion.

How do I recover deleted configurations?

Deleted configurations cannot be recovered directly. Contact your system administrator for database recovery options or recreate the configurations with new entries.

Troubleshooting

Cannot delete a configuration option

This means the configuration is tied to existing records. Check which entities are using it and update or delete those first. Contact the system administrator if needed.

Users cannot log in after configuration changes

Verify that the user's role still has appropriate permissions assigned. Check that no mandatory settings were removed that the user's workflow depends on.

Cannot create new soldiers after settings changes

Ensure all mandatory configuration items (ranks, companies) still exist. Check system logs for error messages. Verify your user has permission to modify soldier records.
