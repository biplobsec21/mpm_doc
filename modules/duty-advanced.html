Advanced Duty Assignments

Deep technical guide for assignment algorithm

🔥 Expert Content

This guide is designed for system administrators, developers, and advanced users who need to understand the complex duty assignment algorithm and troubleshoot complex scenarios.

🎯 Module Overview

This advanced guide explains the most complex module of the Manpower Management System: the duty assignment algorithm that handles automatic soldier selection, conflict detection, and fair rotation.

Key Topics Covered:

  • Exclusion list building (7 types)
  • Multi-layer filtering process (8 sequential filters)
  • Time overlap conflict detection
  • Fair rotation enforcement
  • Multi-day and overnight duty calculations
  • Group vs individual duty differentiation
  • Real-world scenarios with 9+ detailed examples

Assignment Algorithm Architecture

Part 1: Exclusion List Building

The system builds an exclusion list by checking 7 different categories:

  • 1. Soldiers on leave
  • 2. Soldiers on medical status
  • 3. Soldiers in training/cadre
  • 4. Soldiers with prior fixed duties
  • 5. Soldiers in service/deployment
  • 6. Soldiers with appointments (if conflicting)
  • 7. Soldiers with previous roster assignments

Part 2: Multi-Layer Filtering

8 sequential filters are applied to narrow down available soldiers:

  1. Filter 1: Exclude by status
  2. Filter 2: Exclude by availability
  3. Filter 3: Exclude by rank match (if specified)
  4. Filter 4: Exclude by company match (if specified)
  5. Filter 5: Exclude by prior assignments
  6. Filter 6: Exclude by time conflicts
  7. Filter 7: Apply fair rotation
  8. Filter 8: Randomly select from remaining pool

Part 3: Time Conflict Detection

The system detects overlapping time periods: Any duty that shares even 1 minute with another duty/leave/training is flagged as a conflict. Handles 24-hour duties and overnight assignments.

Part 4: Fair Rotation

The system tracks how many duties each soldier has been assigned and prefers soldiers with fewer assignments to maintain fair distribution.

Common Complex Scenarios

Scenario 1: Overnight Duty Conflict

Duty: 22:00 - 06:00 next day. System must check both days for conflicts. Soldier with leave on next day would be excluded even though duty starts before their leave.

Scenario 2: Multi-Day Roster

5-day duty with rotating soldiers. System must track which soldiers are assigned which days and prevent same soldier being assigned same day twice.

Scenario 3: Training Conflict

Soldier enrolled in 3-day course. Any duty overlapping any day of course is excluded. Must check all training types (courses, cadre, exercises).

Troubleshooting Algorithm Issues

Why are qualified soldiers being excluded?

Check the exclusion list. Soldiers might have overlapping assignments, leave, training, or appointments. Review each exclusion category.

Why is fair rotation not working?

Fair rotation is applied AFTER all filters. If very few soldiers pass all filters, same soldiers may be selected repeatedly.

How are time conflicts calculated?

Any overlap of 1+ minute is a conflict. Overnight duties span two days. Always check both start and end dates.
